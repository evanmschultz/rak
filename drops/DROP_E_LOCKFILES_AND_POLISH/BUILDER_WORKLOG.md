- **Mage targets run:** `mage test` (RED: compile failure — undefined `ErrNotRegularFileOrDirectory`), `mage test` (GREEN: all 9 packages pass), `mage build` (pass)
- **Notes:**
  - The existing `Detect` code already called `os.Stat` and handled the regular-file fast-path (`info.Mode().IsRegular()`). The guard was inserted immediately after that block: `if statErr == nil && !info.Mode().IsDir()` catches all non-regular non-directory modes (devices, sockets, named pipes, irregular files). When stat fails, the path falls through to the git probe as before.
  - Error shape: `fmt.Errorf("lister: detect: %s: %w", absRoot, ErrNotRegularFileOrDirectory)` — uses `%w` so callers can `errors.Is(err, lister.ErrNotRegularFileOrDirectory)`. The path in the message is `absRoot` (already resolved via `EvalSymlinks`) for clarity.
  - Test uses `/dev/null` (always a character device on macOS/Linux) per PLAN.md spec. Guards with `os.Stat("/dev/null")` to skip on platforms where it is absent.
  - Verified via QA Falsification: `mode&os.ModeType == 0` for regular files, so the guard never triggers for them. Directories have `mode.IsDir() == true`, so guard also never triggers. Only non-regular non-directory modes (ModeDevice, ModeCharDevice, ModeNamedPipe, ModeSocket, ModeIrregular) pass the guard.
